# Спецификация: Лексическая структура языка Veo

**Статус:** Черновик (stage1)\
**Область применения:** Компилятор `veoc`, фаза лексического анализа

Настоящий документ определяет правила деления исходного кода языка Veo на минимальные значимые единицы - токены (лексемы).

## 1. Входной поток и кодировка

Компилятор принимает на вход поток байт. На этапе лексического анализа валидными признаются символы стандартной таблицы
**ASCII**. Любые неформатные байты, не входящие в правила токенизации, приводят токенизатор в состояние `TOK_UNKNOWN`.

## 2. Пробельные символы и комментарии

Пробельные символы и комментарии служат разделителями токенов и полностью игнорируются лексером (не передаются в парсер).

### 2.1 Пробельные символы

Игнорируются стандартные ASCII-пробелы, символы табуляции, перевода строки и возврата каретки (`\t`, `\n`, `\r`, ` `).

### 2.2 Комментарии

Язык поддерживает два типа комментариев:

- **Однострочные комментарии:** Начинаются с символов `//` и продолжаются до первого символа новой строки (`\n`) или
конца файла (`EOF`).

- **Многострочные комментарии:** Начинаются с `/*` и продолжаются до закрывающей последовательности `*/`.
Вложенность многострочных комментариев текущей версией лексера не поддерживается.

## 3. Идентификаторы и Ключевые слова

### 3.1 Идентификаторы (`TOK_ID`)

Идентификатор используется для именования переменных, функций, типов и модулей.

- **Правило начала:** Должен начинаться с латинской буквы (`a-z`, `A-Z`), знака подчеркивания `_` или символа
коммерческого `@`.
- **Последующие символы:** Могут содержать латинские буквы (`a-z`, `A-Z`), цифры (`0-9`), знаки подчеркивания `_`
и символы `@`.

### 3.2 Ключевые слова

Если выделенный идентификатор совпадает с одной из строк в таблице ниже, токен классифицируется как **ключевое слово**.

|           Ключевое слово           |         Тип токена       |              Описание конструкции             |
|------------------------------------|--------------------------|-----------------------------------------------|
|          `bool`, `char`            |  `TOK_BOOL`, `TOK_CHAR`  |           Примитивные типы данных             |
| `i8`, `i16`, `i32`, `i64`, `isize` | `TOK_I8` ... `TOK_ISIZE` |         Знаковые целочисленные типы           |
| `u8`, `u16`, `u32`, `u64`, `usize` | `TOK_U8` ... `TOK_USIZE` |       Беззнаковые целочисленные типы          |
|           `f32`, `f64`             |   `TOK_F32`, `TOK_F64`   |           Типы с плавающей точкой             |
|               `let`                |         `TOK_LET`        |            Объявление переменной              |
|              `const`               |        `TOK_CONST`       |            Объявление константы               |
|              `func`                |        `TOK_FUNC`        |             Объявление функции                |
|              `return`              |         `TOK_RET`        |        Оператор возврата из функции           |

## 4. Литералы

### 4.1 Строковые и символьные литералы

- **Строковый литерал (`TOK_STR_LIT`):** Последовательность символов, заключенная в двойные кавычки (`"..."`).
Лексер считывает все символы до закрывающей кавычки. Недостижение закрывающей кавычки до конца файла (`EOF`) является
лексической ошибкой.

- **Символьный литерал (`TOK_CHAR_LIT`):** Последовательность символов, заключенная в одинарные кавычки (`'...'`).
В текущей реализации stage1 лексер принимает содержимое между `'` как сырой байт.

### 4.2 Числовые литералы

Лексер поддерживает целочисленные литералы в четырех системах счисления, литералы с плавающей точкой,
а также явные суффиксы типов.

#### Системы счисления (Префиксы)

- **Двоичная (base 2):** Префикс `0b` или `0B`. Разрешенные символы: `0`-`1`.
- **Восьмеричная (base 8):** Префикс `0o` или `0O`. Разрешенные символы: `0`-`7`.
- **Шестнадцатеричная (base 16):** Префикс `0x` или `0X`. Разрешенные символы: `0`-`9`, `a`-`f`, `A`-`F`.
- **Десятичная (base 10):** Без префикса. Разрешенные символы: `0`-`9`.

#### Особенности разбора

- **Знак минуса:** Если символ `-` идет непосредственно перед цифрой (без пробела), он токенизируется как
часть числового литерала.
- **Разделитель:** Символ подчеркивания `_` разрешен внутри литерала для улучшения читаемости цифр и
игнорируется при вычислении значения.
- **Разделитель дробной части:** Точка `.` переводит десятичный литерал в категорию чисел с плавающей точкой,
если за ней следует цифра. В не-десятичных системах счисления точка запрещена и прерывает разбор токена.

#### Суффиксы типов

Литерал может завершаться явным указанием типа. В этом случае значение токена обрезается
(суффикс исключается из строкового значения val), а токен получает специализированный тип:

- **Знаковые целые:** `i8`, `i16`, `i32`, `i64` -> соответствующие токены `TOK_I*_LIT`. Суффикс `iz`
генерирует `TOK_ISIZE_LIT`.
- **Беззнаковые целые:** `u8`, `u16`, `u32`, `u64` -> соответствующие токены `TOK_U*_LIT`.
Суффикс `uz` генерирует `TOK_USIZE_LIT`.
- **Числа с плавающей точкой:** `f32`, `f64` -> токены `TOK_F32_LIT` / `TOK_F64_LIT`.

Если суффикс отсутствует, токен классифицируется как `TOK_F64_LIT` (при наличии точки) или `TOK_INT_LIT`
(целое число по умолчанию).

## 6. Системные токены

- `TOK_UNKNOWN`: Генерируется при встрече любого недетерминированного байта, который не может быть сопоставлен ни
с одним правилом выше.
- `TOK_EOF`: Конец файла. Сигнализирует парсеру об успешном завершении этапа лексического анализа потока данных.
